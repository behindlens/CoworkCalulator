การใช้งานแบบ Usage-based (Copilot Credit)

เกณฑ์การแบ่งระดับงาน (อ้างอิงแนวทางของ Microsoft): ต้นทุนต่องานของ Copilot Cowork ขึ้นกับ 4 ปัจจัยหลักร่วมกัน คือ (1) โมเดลที่ agent เลือกใช้ตอบ (เล็ก/กลาง/reasoning) (2) ปริมาณ context จากองค์กรที่ต้องดึงมาอ้างอิง เช่น ไฟล์ อีเมล แชท (3) จำนวน tool/connector ที่ถูกเรียกใช้ในงานนั้น และ (4) เวลา/จำนวนขั้นตอนที่ agent ใช้จนงานเสร็จ

งานเบา (light, $1–3): ใช้โมเดลพื้นฐาน ไม่ต้องดึง context จากหลายแหล่ง ทำเสร็จในขั้นตอนเดียวหรือไม่กี่ขั้นตอน เช่น สรุปอีเมล/ข้อความสั้น ตอบคำถามทั่วไป แก้ไข-ขัดเกลาข้อความ แปลภาษาสั้นๆ
งานปานกลาง (medium, $4–7): ต้องดึงข้อมูลจากหลายแหล่งในองค์กร (ไฟล์ อีเมล Teams) และ/หรือเรียก tool 2–3 ตัวร่วมกัน เช่น วิเคราะห์ไฟล์ Excel ขนาดกลาง สร้าง draft สไลด์/เอกสาร ค้นหาและสรุปข้อมูลจากหลายเอกสาร
งานหนัก (heavy, $7+): งานแบบ multi-step ที่ต้องใช้ reasoning เชิงลึก เรียก tool หลายตัวหลายรอบ หรือ orchestrate การทำงานข้าม agent เช่น จัดทำรายงานฉบับเต็มจากหลายแหล่งข้อมูล งานวิจัยเชิงลึก (deep research) หรือสร้าง workflow อัตโนมัติที่ซับซ้อนหลายขั้นตอน

ตั้งค่าการซื้อ Copilot Credit

เลือกวิธีจ่ายเงินของ Copilot Studio และระบุว่าผู้ใช้งานมี M365 Copilot license หรือไม่

การใช้งานทั้งหมดด้านล่างถูกรวมอยู่ใน Microsoft 365 Copilot license แล้ว (ภายใต้ fair-use limit) จึงไม่มีการหัก Copilot Credit เพิ่ม
วิธีคิด Copilot Credit: Credits = (จำนวนใช้งาน ÷ หน่วยฐาน) × Rate
สำหรับแถว AI tool: Tokens = จำนวนงาน/เดือน × สมมติฐาน tokens/งาน (ปรับสมมติฐานได้ต่อแถว) แล้วนำ Tokens ไปแทนค่า "จำนวนใช้งาน" ในสูตรด้านบน
แปลงเป็นเงิน: Cost (USD) = Credits × ราคา/Credit → Cost (THB) = Cost (USD) × อัตราแลกเปลี่ยน

ตารางอัตราคิด Copilot Credit ต่อ Feature

ใส่ปริมาณการใช้งานต่อเดือนของแต่ละ feature เพื่อดูจำนวน Credit และค่าใช้จ่ายที่เกิดขึ้นจริง — แถว AI tool ใส่เป็น "จำนวนงาน/เดือน" แล้วระบบแปลงเป็น token ให้อัตโนมัติ

Feature หน่วยคิดเงิน Rate (Credit) ปริมาณ/เดือน Tokens ที่ใช้ (ประมาณ) Credits USD THB
